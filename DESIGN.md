# Perceptron-3 衔尾蛇架构设计

## 1. 核心概念

### 1.1 层定义

Layer 0 为输出层。以4层网络为例，正向传播路径为：输入 → Layer 3 → Layer 2 → Layer 1 → Layer 0。

各层的输入关系为：
- Layer 0 的输入来自 Layer 1
- Layer 1 的输入来自 Layer 2
- Layer 2 的输入来自 Layer 3
- Layer 3 的输入来自外部输入

### 1.2 引擎游走规则

系统包含多个计算引擎。所有引擎沿正向方向行走，从最深层向输出层移动，形成循环：Layer 3 → Layer 2 → Layer 1 → Layer 0 → Layer 3。


以2引擎4层为例，时间步与层分配的关系为：
- 时间步0：Engine 0处理Layer 3，Engine 1处理Layer 1
- 时间步1：Engine 0处理Layer 2，Engine 1处理Layer 0
- 时间步2：Engine 0处理Layer 1，Engine 1处理Layer 3
- 时间步3：Engine 0处理Layer 0，Engine 1处理Layer 2
- 时间步4：回到时间步0的分配，形成循环

### 1.3 信息流规则

当引擎处理某一层时，执行以下操作：

1. 使用前向前向传播算法更新该层权重
2. 计算该层期望的输入值，即该层的输入层应该有的激活值
3. 如果输入层是网络内部层，将期望信号存入该输入层的内存池
4. 如果输入层是外部输入，则丢弃期望信号

各层的信号流向为：
- 处理Layer 0时，期望信号存入Layer 1的内存池
- 处理Layer 1时，期望信号存入Layer 2的内存池
- 处理Layer 2时，期望信号存入Layer 3的内存池
- 处理Layer 3时，期望信号丢弃

必须遵循读写分离规则：某一时间步存入内存池的信号，只能在下一时间步读取，不能在同一时间步读取。

周期循环，形成正向游走构建反向传播的信息流动。

## 3. 组件设计

### 3.1 内存池

内存池负责存储各层的期望信号, 在这种设计下, 只用为每一层存一个最新的期望数据

### 3.2 调度器

调度器负责在每个时间步为每个引擎分配处理层。调度器维护当前时间步，根据分配规则返回各引擎的层分配，并提供时间步前进的功能。

### 3.3 引擎

每个引擎绑定到一个网络实例，可以被分配到不同层进行处理。引擎支持在当前层进行前向传播、使用前向前向传播算法更新当前层权重，以及反向投影计算期望的输入值。

反向投影通过伪逆矩阵求解，给定当前层期望的输出值，计算需要的输入值。

## 4. 训练流程

训练过程包含以下步骤：

1. 初始化阶段
   - 创建网络实例
   - 创建指定数量的衔尾蛇引擎
   - 创建调度器
   - 创建内存池
   - 创建前向前向传播更新器

2. 主训练循环
   - 对每个训练样本
     - 对正样本进行完整前向传播，记录各层激活
     - 生成负样本并进行完整前向传播，记录各层激活
     - 执行多个时间步的衔尾蛇更新
       - 从调度器获取当前时间步的层分配
       - 对每个引擎
         - 从对应层的内存池读取期望信号（来自上一时间步）
         - 使用期望信号和正、负样本激活更新该层权重
         - 计算该层期望的输入值
         - 将期望信号存入输入层的内存池
       - 调度器前进到下一时间步

3. 约束条件
   - 读写分离：当前时间步存入的信号，只能在下一时间步读取
   - 信号流向：处理Layer i时，期望信号存入Layer i的输入层的内存池
   - Layer 3的期望信号丢弃

## 5. 设计目标

衔尾蛇架构的设计目标是通过让多个引擎在不同层之间正向游走，形成信息的反向流动，从而实现类似反向传播的效果，同时保持前向前向传播的简洁性。这种设计旨在同时降低时间复杂度和空间复杂度，通过在GPU中并行处理提高训练效率。

这只是一个试验, 所以不建议使用多线程和显卡api实现
